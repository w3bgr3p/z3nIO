

Центральная страница управления ZP7-задачами. Показывает все задачи из БД, позволяет запускать команды, менять настройки и смотреть логи — без открытия ZennoPoster.

Путь: `?page=zp7`

---

## Список задач

Левая панель. Задачи сгруппированы по машине, внутри — по проекту (префикс имени до первой точки).

Каждая строка показывает:

- цветная точка — активна ли задача прямо сейчас (`IsWork`)
- имя задачи
- статус: `Perform`, `Schedule`, `Stop`, `Fail`, `Complete`
- счётчик Done / Total

Группу машины или проекта можно свернуть/развернуть кликом по заголовку. Состояние сохраняется между обновлениями.

**AUTO ON / OFF** — кнопка в шапке. При включённом AUTO список и очередь команд обновляются каждые 3 секунды.

**↺ Refresh** — принудительное обновление списка.

## Фильтры и теги

Панель фильтров над списком задач.

|Элемент|Что фильтрует|
|---|---|
|Поле поиска|Имя задачи (подстрока)|
|Status|`Perform`, `Schedule`, `Stop`, `Fail`, `Complete`|
|All / Working / Idle|Только активные (`IsWork = True`) или неактивные|
|Chips тегов|GroupLabels задачи — один или несколько тегов одновременно|

Чипы тегов появляются автоматически из полей `GroupLabels` всех задач. Активный тег подсвечивается синим. Несколько активных тегов работают как AND — задача должна содержать все выбранные теги.

Кнопка **✕ clear** — сбросить все активные теги.

Кликнув на тег в карточке задачи (Task Detail) — тег добавляется в фильтр.

## Task Detail

Открывается при выборе задачи в списке. Повторный клик по задаче — закрывает.

Показывает:

- имя и путь к файлу проекта
- секция **Execution**: статус, IsWork, потоки, Done, Total, Priority
- секция **Scheduler**: IsActive, Next Run, Last Run, Period, Repeat
- секция **Tags**: список GroupLabels, каждый кликабелен (добавляет в фильтр)

При открытии Task Detail автоматически раскрываются панели [[#Logs панель]] и [[#HTTP панель]].

## Команды

Кнопки в Task Detail отправляют команды в очередь `_commands`. ZP-шаблон опрашивает очередь и выполняет команду.

|Кнопка|Команда|Поведение|
|---|---|---|
|**▶ Start**|`start`|Запустить задачу|
|**■ Stop**|`stop`|Остановить — ждёт завершения текущей итерации|
|**⚡ Interrupt**|`interrupt`|Прервать немедленно, без ожидания|
|**↺ Apply**|`update_settings`|Применить изменённые настройки без перезапуска|
|**+ Tries**|`add_tries`|Прибавить число попыток к счётчику (вводится в диалоге)|
|**= Tries**|`set_tries`|Перезаписать счётчик попыток (вводится в диалоге)|
|**⚙ Threads**|`set_threads`|Задать лимит потоков (вводится в диалоге)|
|**✓ Clear Done**|`clear_success`|Сбросить счётчик успешных выполнений|
|**✗ Clear Fails**|`clear_fails`|Сбросить счётчик неудач|

Статус каждой команды виден в [[#Commands Queue]].

## Settings

Кнопка **⚙ Settings** в Task Detail. Открывает редактор полей `InputSettings` из XML-конфига ZP-задачи.

Поля разбиты по вкладкам (если в XML есть `Tab`-элементы). Поддерживаемые типы полей: текст, число, пароль, boolean (чекбокс), dropdown (выпадающий список), multiselect (чекбоксы).

**💾 Save & Apply** — сохраняет изменения и отправляет команду `update_settings`.

**📋 Copy Payload** — копирует текущие значения полей как JSON-payload для [[shift + ctrl + 1.    z3n8 (Scheduler)]]. Используется чтобы передать настройки задачи в Scheduler-скрипт без ручного ввода.

## Execution Settings

Кнопка **⚙ Execution** в Task Detail. Управляет параметрами выполнения задачи в ZP.

|Поле|Описание|
|---|---|
|Limit of Threads|Максимальное число параллельных потоков|
|Priority|Приоритет задачи в ZP|
|Done Successfully|Счётчик успехов (можно задать вручную)|
|Done All|Общий счётчик попыток|
|Number of Tries|Количество попыток на аккаунт|
|Max Success Stop|Остановить задачу при достижении N успехов|
|Max Fail Stop|Остановить задачу при достижении N ошибок|
|Status|`Newbie`, `Stop`, `Schedule`, `Perform`, `Complete`, `Fail`|
|Proxy|Режим использования прокси|
|Group Labels|Теги задачи через запятую (используются в фильтрах)|

**💾 Apply** — отправляет команду `set_execution_settings` с новыми значениями.

## Scheduler Settings

Кнопка **🕐 Scheduler** в Task Detail. Управляет встроенным планировщиком ZP7.

|Поле|Описание|
|---|---|
|Is Active|Включить / выключить расписание (тоггл)|
|Execute Period|`OneTime`, `EveryDay`, `EveryWeek`, `EveryMonth`, `OnDemand`|
|Intervals|Временные интервалы запуска. Пример: `09:00, 12:00-17:00`|
|Start Date Type|`Immediately` — запустить сразу, `OnDate` — с указанной даты|
|Start Date|Дата и время первого запуска|
|Repeat Type|`Continued`, `ContinuedWithPause`, `Regularly`, `Count`|
|Attempts Range|Зависит от Repeat Type: пауза в минутах / интервал / число попыток|
|End Date Type|`Infinity`, `OnDate`, `Count`|
|End Date|Дата завершения|

**💾 Apply** — отправляет команду `set_scheduler_settings`.

## Logs панель

Появляется при выборе задачи. Показывает логи конкретной задачи в реальном времени через SSE.

При открытии загружается история (последние N записей), затем новые записи добавляются автоматически.

**Фильтры:**

|Элемент|Описание|
|---|---|
|Level|`DEBUG`, `INFO`, `WARNING`, `ERROR`|
|Лимит|Количество загружаемых записей (10–500)|
|**↺**|Перезагрузить историю с текущими фильтрами|
|**🗑**|Удалить все логи этой задачи|

Полный просмотр логов по всем задачам — [[shift + ctrl + 4.    Logs]].

## Commands Queue

Нижняя правая панель. Показывает историю отправленных команд.

Каждая строка: статус, действие, задача, результат / payload, время.

**Статусы:**

|Статус|Описание|
|---|---|
|`pending`|Команда отправлена, ZP ещё не забрал|
|`done`|ZP выполнил команду|
|`error`|Ошибка при выполнении|

**Фильтр** — выбрать `All`, `Pending`, `Done`, `Error`.

**✗ Done** — удалить выполненные команды.

**✗ All** — удалить все команды из очереди.

## Heatmap

Отображается когда ни одна задача не выбрана. Показывает историю обработки аккаунтов по проектам.

Каждая ячейка — один аккаунт. Цвет показывает результат последней обработки и давность:

|Цвет|Значение|
|---|---|
|Зелёный (яркий)|Успех сегодня|
|Красный (яркий)|Ошибка сегодня|
|Зелёный (тёмный)|Успех вчера / 2 дня / 3+ дней|
|Красный (тёмный)|Ошибка вчера / 2 дня / 3+ дней|
|Пустая|Аккаунт ещё не обрабатывался|

Наведение на ячейку — tooltip с номером аккаунта, проектом, временем, результатом и текстом ошибки.

Карточка слева от ячеек показывает: название проекта, прогресс-бар, счётчики успех / ошибка, min/max/avg время обработки, процент успеха.

**↺** в заголовке — принудительно обновить данные (по умолчанию кешируются на 5 минут).

## Edge cases

**Команда зависла в `pending`** — ZP-шаблон не опрашивает очередь или задача не запущена. Убедись что шаблон запущен и поддерживает команды z3n8.

**`stop` не помогает, задача продолжает работать** — `stop` ждёт конца итерации. Если итерация зависла — используй **⚡ Interrupt**.

**Settings открываются пустыми** — у задачи нет `InputSettings` в XML-конфиге, или ZP недоступен по адресу из конфига.

**Теги не появляются в фильтр-баре** — поле `GroupLabels` пустое у всех задач. Теги задаются через **⚙ Execution → Group Labels**.

**Heatmap показывает «No project data»** — в БД нет записей истории обработки. Данные появляются после первого завершения аккаунтов в ZP-шаблонах.

**AUTO ON замедляет браузер** — при большом числе задач (500+) можно выключить AUTO и обновлять вручную через **↺ Refresh**.
