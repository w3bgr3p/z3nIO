

Техническое описание системы шифрования данных кошельков в z3nIO / z3n7.


---

## Компоненты

|Компонент|Расположение|Назначение|
|---|---|---|
|`safu.key`|`AppContext.BaseDirectory/safu.key` (z3nIO)|32 случайных байта. Генерируется один раз при первой настройке сервера.|
|`jvars.dat`|Путь из `Config.SecurityConfig.JVarsPath`|AES-CBC blob с JSON `{ "cfgPin": "..." }`. Зашифрован HWID машины.|
|`_wlt`|PostgreSQL / SQLite|Зашифрованные ключи кошельков (secp256k1, base58, bip39).|

---

## Схема деривации ключей

```
// jvars.dat (HWID-only, без PIN)
salt = HMAC-SHA256(safu.key, "hwid-only")[..16]
key  = PBKDF2-SHA256(HWID, salt, 100_000, 32)

// Ключи кошельков в _wlt
salt = HMAC-SHA256(safu.key, "{HWID}:{acc}")[..16]
key  = PBKDF2-SHA256(pin, salt, 100_000, 32)

// Формат AES blob: IV(16) + Ciphertext + HMAC-SHA256(IV+Ciphertext)(32)
```

Соль вычисляется через `safu.key` — без ключа даже знание HWID и PIN не позволяет расшифровать данные.

## Мультимашинная схема

```
Сервер:
  safu.key ────────────────────────────────────────────→ клиент (копия)
  EncryptHWIDOnly(jVarsJson, clientHwid) → jvars.dat  → клиент (персональный)
  _wlt: Encode(key, pin, acc) с HWID_сервера           → БД (общая)

Клиент:
  DecodeHWID(jvars.dat)       → pin    (safu.key + HWID_клиента)
  Decode(_wlt[acc], pin, acc) → key    (safu.key + HWID_сервера зашит в соль)
```

Ключи кошельков в БД зашифрованы HWID **сервера**. Клиент расшифровывает их потому что имеет тот же `safu.key` — он используется при деривации соли, которая включает HWID сервера.

## z3nIO — серверная часть

### Ключевые файлы

|Файл|Описание|
|---|---|
|`Safu.cs`|Криптографическое ядро|
|`InternalTasks.cs`|Загрузка `jvars.dat` при старте, property `_jVarsPath`|
|`InternalTasks.GenerateClientBundle.cs`|Генерация бандла для клиента|
|`ConfigHandler.cs`|`POST /config/jvars` — запись PIN и пути|

### Эндпоинты

|Метод|Путь|Описание|
|---|---|---|
|`POST`|`/config/jvars`|Принимает Base64(JSON{pin, jVarsPath, localVars}). Шифрует и записывает `jvars.dat`.|
|`POST`|`/import/wallets`|`{ "type": "seed\|evm\|sol", "lines": [...] }`. Шифрует каждый ключ через `SAFU.Encode(key, pin, acc)`.|
|`POST`|`/import/structure`|Создаёт таблицы из `db_template.json`. PIN не требуется.|

### GenerateClientBundle — payload

```json
{
  "clientHwid": "base64-строка",
  "clientName": "worker01",
  "outputFolder": "C:\\bundles"
}
```

Результат в `outputFolder/worker01/`: `jvars.dat` (зашифрован HWID клиента) + `safu.key` (копия серверного).

`outputFolder` опционален. Fallback: `AppContext.BaseDirectory/clients/{clientName}/`.

## z3n7 — ZennoPoster lib

### Инициализация

```csharp
public void InitVariables(string author = "")
{
    _SAFU();                                                           // читает jvars.dat → project.Var("jVars")
    SAFU.InitZ3n8(_project, _project.Path + "\\.internal\\safu.key"); // регистрирует Z3n8SAFU в FunctionStorage
}

private void _SAFU()
{
    string jFile = _project.Var("jVars");          // переменная содержит ПУТЬ к файлу
    _project.Var("jVars", File.ReadAllText(jFile)); // заменяется на зашифрованный blob
}
```

После `InitZ3n8` все операции шифрования используют z3nIO-совместимую криптографию через `FunctionStorage`.

### Публичный API

```csharp
// Получить значение из jVars по ключу
string pin = project.SecureVar("cfgPin");

// Зашифровать / расшифровать ключ кошелька
string blob = SAFU.Encode(project, plaintext);  // использует pin + acc из project
string key  = SAFU.Decode(project, blob);

// HWID-only операции
string blob = SAFU.EncryptHWID(project, plaintext);
string text = SAFU.DecryptHWID(project, blob);

// Детерминированный пароль профиля из HWID
string pass = SAFU.HWPass(project);
```

### FunctionStorage — зарегистрированные функции

|Ключ|Сигнатура|
|---|---|
|`SAFU_Encode`|`(project, toEncrypt, pin, acc) → string`|
|`SAFU_Decode`|`(project, toDecrypt, pin, acc) → string`|
|`SAFU_HWPass`|`(project, pin, acc) → string`|
|`SAFU_EncryptHWID`|`(project, toEncrypt) → string`|
|`SAFU_DecryptHWID`|`(project, toDecrypt) → string`|

### Совместимость с .NET 4.8

z3n7 таргетит `net48`. PBKDF2-SHA256 реализован вручную — в .NET 4.8 `Rfc2898DeriveBytes` не принимает `HashAlgorithmName`. Результат побайтово идентичен z3nIO (`net8`, `Rfc2898DeriveBytes` + `HashAlgorithmName.SHA256`).

## Миграция со старой схемы

Используется одноразовая задача `SafuMigrate`:

1. Читает `_wallets` (старая таблица, legacy-схема: хардкод `HWID_ONLY_SALT_V1`, SHA-1 PBKDF2).
2. Расшифровывает legacy-схемой.
3. Перешифровывает новой схемой (safu.key, SHA-256 PBKDF2).
4. Пишет в `_wlt`.
5. Перезаписывает `jvars.dat` в новый формат.

После успешной миграции удали `SafuMigrate.cs` и `InternalTasks.Migrate.cs` из проекта.
