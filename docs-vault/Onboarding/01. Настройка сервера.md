

Первичная настройка z3nIO на серверной машине. Выполняется один раз.

**Что потребуется:**

- z3nIO запущен
- Доступ к `http://localhost:{port}/?page=config`
- Придуманный Wallet PIN

---

## Шаг 1 — Открыть Config

При первом запуске z3nIO без `appsettings.secrets.json` дашборд сам откроется на странице [[shift + ctrl + 0.    Config]].

Если файл уже есть — перейди вручную: `?page=config`.

---

## Шаг 2 — Настроить DbConfig

В секции **DbConfig** выбери режим и заполни поля.

**SQLite** — если ферма работает на одной машине:

|Поле|Пример|
|---|---|
|SQLite Path|`C:\z3nIO\data\farm.db`|

**Postgre** — если к БД подключаются несколько клиентских машин:

|Поле|Пример|
|---|---|
|PG Host|IP сервера БД|
|PG Port|`5432`|
|PG Database|`postgres`|
|PG User|`postgres`|
|PG Password|пароль|

Нажми **💾 Save Config**.

---

## Шаг 3 — Настроить Logs & Server

Заполни адреса, на которые ZP-шаблоны будут отправлять данные:

|Поле|Значение|
|---|---|
|Dashboard Port|`10993` (или другой свободный порт)|
|Log Host|`http://localhost:10994`|
|Traffic Host|`http://localhost:10995`|
|Logs Folder|`./logs`|

Нажми **💾 Save Config**.

---

## Шаг 4 — Настроить jVars (Security)

jVars — зашифрованное хранилище PIN для расшифровки кошельков. `safu.key` генерируется автоматически при первом сохранении.

1. В секции **Security · jVars** введи **Wallet PIN** — произвольный пароль, который будет ключом шифрования кошельков.
2. Укажи **jVars File Path** — путь где будет храниться зашифрованный файл. Пример: `C:\z3nIO\keys\vars.dat`
3. Нажми **🔐 Set PIN & Path**.

После сохранения:

- Файл `vars.dat` создан и зашифрован привязкой к HWID этой машины.
- Файл `safu.key` создан в папке приложения z3nIO.
- PIN не сохраняется в конфиг и не отображается при следующей загрузке.

> Запомни PIN или сохрани в надёжном месте. Без PIN невозможно расшифровать кошельки и перегенерировать бандлы для клиентов.

---

## Шаг 5 — Создать структуру БД

Перейди в [[shift + ctrl + 0.    Config#Fill DB (Import)]]. Нажми **⬆ Fill DB** и выбери `Structure`.

Это создаст все необходимые таблицы в БД (скачивается шаблон с GitHub). PIN не требуется.

---

## Шаг 6 — Импортировать данные фермы

Через **⬆ Fill DB** загрузи данные в следующем порядке:

1. **Wallets** — приватные ключи или seed-фразы. Шифруются через PIN.
2. **Proxy** — прокси-строки.
3. **Addresses** — публичные адреса (если используются отдельно от кошельков).
4. **Social** — аккаунты Twitter, Discord и т.д.
5. Остальные типы — по необходимости.

Подробнее о форматах импорта: [[shift + ctrl + 0.    Config#Fill DB (Import)]].

---

## Edge cases

**`safu.key` уже существует** — повторное нажатие **🔐 Set PIN & Path** не пересоздаёт `safu.key`. Ключ генерируется один раз при первоначальной настройке.

**PIN забыт** — `jvars.dat` необходимо пересоздать с новым PIN. Все ранее зашифрованные кошельки в БД станут нечитаемы. Потребуется повторный импорт кошельков.

**БД не подключается после Save** — проверь `Config → Status → DB Mode` и `DB Host`. Убедись что PostgreSQL запущен и доступен по указанному адресу.
