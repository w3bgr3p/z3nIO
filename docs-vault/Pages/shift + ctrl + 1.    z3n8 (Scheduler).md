

Встроенный планировщик задач. Запускает скрипты (.py, .js, .exe, .bat и др.) по расписанию — cron, интервалу или фиксированному времени.

Путь: `/scheduler.html`

Открывается по умолчанию при старте z3nIO (если конфиг уже настроен).

---

## Список задач

Левая панель. Задачи с одинаковым префиксом имени (до первой точки) группируются в раскрываемую папку. Одиночные задачи отображаются отдельно ниже групп.

Каждая строка:

- цветная точка — включена (`enabled`) или выключена
- имя задачи (внутри группы — часть после точки)
- расписание и дата последнего запуска
- статус: `idle`, `running`, `error`, `disabled`
- иконка executor
- счётчики успех / всего

**Поиск** — фильтрует по имени и пути к скрипту.

**Executor filter** — чипы над списком. Появляются если используется более одного типа executor. Клик — показать только задачи с этим executor.

**+ New** — создать новую задачу.

Список обновляется каждые 10 секунд автоматически.

## Создание и редактирование задачи

Вкладка **Settings** в правой панели. Открывается при нажатии **+ New** или выборе задачи → вкладка Settings.

### Основные поля

|Поле|Описание|
|---|---|
|Name|Имя задачи. Точка в имени создаёт группу: `GroupName.TaskName`|
|Script / Task|Путь к скрипту или имя внутренней задачи|
|Executor|Тип исполнителя (см. таблицу ниже)|
|Arguments|Аргументы командной строки|
|Enabled|`Yes` — задача активна и будет запускаться по расписанию|

### Executor

|Значение|Что запускает|
|---|---|
|`python`|Python-скрипт|
|`node`|Node.js-скрипт|
|`ts-node`|TypeScript через ts-node|
|`csx`|C# script через dotnet-script|
|`exe`|Исполняемый файл|
|`bat`|Batch-скрипт|
|`bash`|Bash-скрипт|
|`ps1`|PowerShell-скрипт|
|`internal`|Встроенная задача z3nIO (не для ручного создания)|

### Расписание

Выбери Period — остальные поля появятся автоматически:

|Period|Поля|Пример|
|---|---|---|
|`OnDemand`|—|Только ручной запуск через **▶ Run**|
|`EveryDay`|Time(s)|`09:00` или `09:00, 14:30`|
|`EveryWeek`|Time(s) + дни недели|`09:00`, отметь Mon/Wed/Fri|
|`EveryMonth`|Time(s) + день месяца|`09:00`, день `1`|
|`Interval`|Every (min)|`30` — каждые 30 минут|

Поля **Cron**, **Interval (min)**, **Fixed time** в секции Result — итоговые значения, сгенерированные построителем. Можно заполнить вручную вместо построителя.

> Если заполнить Cron вручную — построитель переключится в `OnDemand`. Это нормально: cron используется как есть.

### Overlap

Что делать если задача ещё выполняется, а расписание снова сработало:

|Значение|Поведение|
|---|---|
|`Skip`|Пропустить новый запуск|
|`Parallel`|Запустить параллельно. Появляется поле **Max threads**|
|`Kill & restart`|Убить текущий процесс и запустить заново|

**Save** — сохранить задачу.

## Execution tab

Открывается при выборе задачи. Показывает текущее состояние выполнения.

|Поле|Описание|
|---|---|
|Status|`idle`, `running`, `error`|
|Is Working|Да / Нет|
|Done|Успешных запусков|
|Total|Всего запусков|
|Failed|Запусков с ненулевым exit code|
|Last Exit|Код завершения последнего запуска (`0` — успех, `-1` — прерван)|
|Last Run|Дата и время последнего запуска|
|Period|Текущее расписание|
|On Overlap|Политика перекрытия|
|Executor|Тип исполнителя|

Если задача **запущена и не параллельная** — появляются поля PID, Uptime, Memory. Обновляются каждые 2 секунды.

Если задача **параллельная** (`on_overlap = parallel`) — вместо PID/Uptime/Memory появляются:

- **Active instances** — список активных параллельных запусков с runId, uptime, памятью и кнопкой **✕** для остановки конкретного инстанса
- **Queue (pending)** — число запусков в очереди ожидания + кнопка **Clear queue**

### Кнопки действий

|Кнопка|Действие|
|---|---|
|**▶ Run**|Запустить немедленно, не ожидая расписания|
|**⏸ Pause / ▶ Resume**|Включить / выключить задачу (`enabled`)|
|**■ Interrupt**|Остановить выполняющийся процесс. Если запущено несколько инстансов — появится диалог выбора|

## Output tab

Вывод последнего запуска скрипта (stdout / stderr).

Показывает: время последнего запуска, exit code, текст вывода.

Если у задачи есть логи через Logger — они отображаются в нижней [[#Logs панель]] вместо Output. Output — запасной вариант для скриптов без логгера.

**↺** — перезагрузить вывод вручную.

**Live badge** — появляется если задача запущена прямо сейчас.

## Payload — Set Values и Build Schema

Payload — это данные конфигурации задачи: схема полей (schema) и их значения (values). Хранится в БД, передаётся скрипту при запуске как аргумент или через переменную окружения.

### ⚙ Set (Values)

Форма для заполнения значений полей задачи. Поля отображаются согласно schema — с типами, лейблами, опциями.

Если schema содержит вкладки (`tab`-поля) — поля разбиты по вкладкам.

**Save** — сохранить значения.

### 🔧 Build (Schema)

Конструктор схемы. Позволяет добавить, удалить и переупорядочить поля.

Каждое поле: **key** (имя переменной), **type**, **label / options**.

|Type|Описание|
|---|---|
|`text`|Текстовое поле|
|`password`|Скрытое поле|
|`boolean`|Чекбокс|
|`select`|Выпадающий список. В поле options: `opt1, opt2, opt3`|
|`multiselect`|Множественный выбор|
|`section`|Разделитель-заголовок (без key)|
|`html`|Произвольный HTML (без key)|
|`tab`|Создаёт новую вкладку в Values|

**+ Field**, **+ Section**, **+ HTML** — добавить элемент.

**Save** — сохранить схему.

### 📥 Import / 📤 Export

**📤 Export** — скопировать payload (schema + values) текущей задачи в буфер обмена как JSON.

**📥 Import** — вставить payload из буфера. Используется для переноса настроек между задачами или из [[shift + ctrl + 2.    ZP7]] (кнопка **📋 Copy Payload** в PM Settings).

Формат payload:

```json
{
  "schema": "[{\"key\":\"mode\",\"label\":\"Mode\",\"type\":\"select\",\"options\":\"fast, slow\"}]",
  "values": "{\"mode\":\"fast\"}"
}
```

## Logs панель

Нижняя левая панель. Показывает логи выбранной задачи. Обновляется через SSE в реальном времени.

Если логов нет — показывает stdout из Output как запасной вариант.

**Фильтры:**

|Элемент|Описание|
|---|---|
|Level|`DEBUG`, `INFO`, `WARNING`, `ERROR`|
|Лимит|Количество записей|
|Only this run|Показать только логи последнего запуска|
|**↺**|Перезагрузить|
|**🗑**|Очистить логи этой задачи|

## HTTP панель

Нижняя правая панель. Трафик выбранной задачи через SSE.

**Фильтры:** Method, Status, URL, лимит, Only this run, **↺**, **🗑**.

## Edge cases

**Задача не запускается по расписанию** — проверь что `Enabled = Yes` и поле расписания заполнено. Для cron используй валидатор (например crontab.guru).

**Статус завис в `running`** — скрипт не завершился. Проверь Output или Logs. Используй **■ Interrupt** для принудительной остановки.

**`error` не сбрасывается** — статус `error` остаётся до следующего успешного запуска. Принудительно сбросить нельзя — запусти задачу вручную через **▶ Run**.

**Executor not found** — Python / Node не в PATH. Проверь что интерпретатор доступен из командной строки. Альтернативно укажи полный путь к интерпретатору в поле Script.

**Parallel + большой Max threads** — каждый инстанс — отдельный процесс. При большом числе потоков следи за памятью через Execution tab → Active instances.

**Import payload — «Missing schema or values»** — payload скопирован не полностью или формат неверный. Убедись что скопирован весь JSON включая поля `schema` и `values`.
