

Дерево copy-paste шаблонов. Клик по листу — содержимое копируется в буфер одним действием.

Путь: `?page=clips`

---

## Дерево

Левая панель. Шаблоны организованы в дерево папок по полю `path`.

Путь формирует вложенность: `/Ethereum/Bridges` создаёт папку `Ethereum` с подпапкой `Bridges`. Шаблон появляется как лист внутри последней папки пути.

Папки открываются / закрываются кликом. Состояние раскрытия сохраняется между сессиями.

**Поиск** — фильтрует по title, path и содержимому. При активном поиске все папки разворачиваются автоматически.

**+ New** — создать новый шаблон.

---

## Копирование

**Клик** по листу — скопировать содержимое шаблона в буфер. Лист на секунду подсвечивается зелёным, появляется toast-уведомление.

Если содержимое пустое — появится ошибка `No content`.

---

## Редактор

Правая панель. Открывается при создании или редактировании шаблона.

**Открыть редактор:**

- Двойной клик по листу
- Правый клик по листу
- Кнопка **+ New** (новый шаблон)

**Поля:**

|Поле|Описание|
|---|---|
|Path|Путь в дереве. Пример: `/Ethereum/Bridges`. Можно оставить пустым — шаблон будет в корне|
|Title|Название листа в дереве|
|Content|Содержимое шаблона — произвольный текст|

**Save** — сохранить. Дерево перестраивается автоматически.

**Cancel** — закрыть редактор без сохранения.

**Delete** — удалить шаблон (с подтверждением). Появляется только при редактировании существующего шаблона.

---

## Edge cases

**Шаблон не появился в дереве после сохранения** — проверь поле Path. Если Path пустой — шаблон попадёт в корень дерева, а не в папку.

**Поиск не находит шаблон** — поиск работает по title, path и content. Убедись что хотя бы одно из полей содержит искомую строку.

**Clipboard access denied** — браузер заблокировал доступ к буферу. Это происходит если страница открыта не через `localhost`. Убедись что дашборд открыт через `http://localhost:{port}`.
