

Перехваченный HTTP-трафик от ZP-задач. Показывает запросы и ответы в реальном времени, позволяет переотправить любой запрос и сгенерировать готовый код.

Путь: `?page=http`

Трафик поступает от ZP-шаблонов через `TrafficHost` из [[shift + ctrl + 0.    Config]]. Новые запросы появляются автоматически через SSE.

---

## Список запросов

Каждая строка:

|Колонка|Описание|
|---|---|
|Time|Время запроса|
|Method|`GET`, `POST`, `PUT`, `DELETE` и др.|
|URL|Адрес запроса (обрезается до 60 символов, полный — в tooltip)|
|Status|HTTP-статус ответа|
|Duration|Время выполнения в мс|
|Account|Аккаунт, с которым работал шаблон|

Цвет статуса: зелёный (2xx), синий (3xx), жёлтый (4xx), красный (5xx).

Клик по строке — открыть [[#Detail-панель]].

**Счётчики в шапке:** Total, Success (% от 2xx), Avg Time.

## Фильтры

|Фильтр|Описание|
|---|---|
|Method|`GET`, `POST`, `PUT`, `DELETE`|
|Status|`2xx`, `3xx`, `4xx`, `5xx`|
|URL|Подстрока URL|
|Machine|Конкретная машина|
|Project|Конкретный проект|
|Лимит|Число загружаемых записей (10–1000, по умолчанию 200)|

**🔄 Refresh** — принудительное обновление.

**⏸ Auto ON/OFF** — включить / выключить SSE live stream.

**✖ Reset** — сбросить все фильтры.

**🗑 Clear Logs** — удалить все HTTP-логи с сервера (с подтверждением).

## Detail-панель

Открывается при клике на строку. Содержит полную информацию о запросе и все инструменты работы с ним.

### Метаданные

URL (клик — копировать), метод, статус, длительность, timestamp.

Бейджи: machine, project, origin, account, session, proxy.

### Request

Раскрываемые секции:

- **Headers** — список заголовков. Клик по ключу — скопировать `Key: Value`. Клик по значению — скопировать только значение.
- **Cookie** — строка cookies (если есть). Клик — копировать.
- **Body** — тело запроса с pretty-print для JSON. Клик — копировать.

### Response

Раскрываемые секции:

- **Headers** — аналогично Request Headers.
- **Body** — тело ответа с pretty-print. Клик — копировать.

## Replay

Кнопка **▶ Play** в Detail-панели. Открывает модальное окно для повторной отправки запроса.

Поля модального окна предзаполнены данными из выбранного запроса:

|Поле|Описание|
|---|---|
|Method|HTTP-метод (редактируемый)|
|URL|Адрес (редактируемый)|
|Headers|Заголовки, по одному на строку в формате `Key: Value` (редактируемые)|
|Body|Тело запроса (редактируемое)|

**▶ Send** — отправить запрос. Результат появляется в поле **Response** справа: статус, время, заголовки ответа, тело ответа.

**📋 Copy response** — скопировать ответ в буфер.

**📋 cURL** — скопировать текущее состояние формы как cURL-команду.

> Replay отправляет запрос через встроенный прокси z3nIO на порту `DashboardPort + 1`. Если этот порт недоступен — Replay не работает.

## Кодогенерация

Кнопки в Detail-панели генерируют готовый код из выбранного запроса. Код копируется в буфер одним кликом.

|Кнопка|Что генерирует|
|---|---|
|**⎘ HttpClient**|C# `HttpClient` с заголовками, cookies и телом|
|**⎘ ZP7**|`ZennoPoster.HTTP.Request(...)` — готовый вызов для ZP-шаблона|
|**⎘ Hybrid**|C# `HttpClient` в синхронном стиле (`.Result`) для ZP C#-кода|
|**⎘ Python**|`requests` с заголовками и телом|
|**⎘ TypeScript**|`fetch` с заголовками и телом|
|**⎘ cURL**|cURL-команда|

### API Skeleton и API Example

Работают не с одним запросом, а со всей текущей выборкой (с учётом активных фильтров). Группируют запросы по уникальным endpoint'ам и формируют JSON-описание API.

|Кнопка|Что в requestBody / responseBody|
|---|---|
|**⎘ API Skeleton**|Типы полей вместо значений (`string`, `number`, `boolean`)|
|**⎘ API Example**|Реальные значения из последнего запроса к каждому endpoint'у|

Формат результата:

```json
[
  {
    "method": "POST",
    "url": "https://api.example.com/v1/auth",
    "statusCode": 200,
    "requestHeaders": { "content-type": "application/json" },
    "responseHeaders": { "content-type": "application/json" },
    "requestCookies": null,
    "responseCookies": null,
    "requestBody": { "login": "string", "password": "string" },
    "responseBody": { "token": "string", "expires": "number" }
  }
]
```

Используется для быстрого документирования API или как основа для написания скрипта.

## Edge cases

**Трафик не появляется** — `TrafficHost` в [[shift + ctrl + 0.    Config]] не совпадает с адресом, на который ZP-шаблон отправляет трафик. Проверь оба значения.

**Replay возвращает «Unreachable»** — порт `DashboardPort + 1` занят или недоступен. Проверь `Config → Status → Listening Ports`.

**Заголовки в кодогенерации обрезаны или отсутствуют** — заголовки, содержащие `***MASKED***`, намеренно исключаются из кода. Это чувствительные данные (токены, ключи), которые z3nIO маскирует перед сохранением.

**Список не обновляется при включённом Auto** — SSE-соединение разорвалось. Переключи Auto OFF → ON для переподключения.

**API Skeleton / Example пустой** — в текущей выборке нет запросов, или все URL заканчиваются на файловое расширение (`.js`, `.css`, `.png`) — такие запросы исключаются из агрегации.
