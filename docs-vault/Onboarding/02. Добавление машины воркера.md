
Процедура выполняется каждый раз при подключении новой клиентской машины к ферме.

**Предусловие:** сервер z3nIO настроен, jVars разблокированы (PIN введён при старте).

---

## Обзор

Каждый клиент получает персональный бандл из двух файлов:

|Файл|Описание|
|---|---|
|`jvars.dat`|Зашифрован HWID конкретного клиента. Уникален для каждой машины.|
|`safu.key`|Копия серверного ключа. Одинаков для всех клиентов.|

Клиент с чужим `jvars.dat` не сможет расшифровать кошельки — файл привязан к железу.

---

## Шаг 1 — Получить HWID клиента



HWID — технический идентификатор железа. Нужен  для генерации твоих персональных файлов доступа. Это не пароль, передавать безопасно.

запусти шаблон exec.z3n7 из папки с программой в режиме GetHW

После выполнения  значение  `hwid` находятся в буфере обмена .


---

## Шаг 2 — Запустить GenerateClientBundle

В [[shift + ctrl + 1.    z3nIO (Scheduler)]] найди задачу `GenerateClientBundle` и открой **⚙ Set**.

Заполни payload:

```json
{
  "clientHwid": "base64-строка от клиента",
  "clientName": "worker01",
  "outputFolder": "C:\\bundles"
}
```

| Поле           | Обязательно | Описание                                                         |
| -------------- | ----------- | ---------------------------------------------------------------- |
| `clientHwid`   | ✅           | HWID клиентской машины                                           |
| `clientName`   | ✅           | Имя для папки с файлами. Пример: `worker01`                      |
| `outputFolder` | ❌           | Куда сохранить бандл. Если не указано — `z3nIO\clients\Roman001\` |

Нажми **▶ Run**.

---

## Шаг 3 — Передать файлы клиенту

После выполнения задачи в `outputFolder\Roman001\` будут два файла:

```
bundles\
  Roman001\
    jvars.dat
    safu.key
```

Оба файла положи в папку `.internal` рядом с `.zp` файлом проекта:

```
МойПроект\
  МойПроект.zp
  .internal\
    jvars.dat
    safu.key
```

---
## Шаг 4 — Настроить переменную проекта

В ZennoPoster открой настройки проекта → вкладка **Переменные**.

Найди переменную `jVars` и задай её значение по умолчанию — **полный путь** к файлу `jvars.dat`:

```
C:\МойПроект\.internal\jvars.dat
```


---

## Шаг 4 — Настроить подключение к базе данных

В ZennoPoster установи глобальные переменные проекта:

в шаблоне exec.z3n7 установи настройки подключения к базе данных. и настройки хостов для отправки логов и траффика

```csharp
project.InitVariables(instance);


var globalVarsToSet = new Dictionary<string, string>();
globalVarsToSet.Add("sqlPgUser","postgres");
globalVarsToSet.Add("sqlPgPass","postgres");
globalVarsToSet.Add("sqlPgHost","localhost");
globalVarsToSet.Add("sqlPgPort","5432");
globalVarsToSet.Add("sqlPgName","postgres");
globalVarsToSet.Add("sqLitePath","");
globalVarsToSet.Add("logHost","http://localhost:38109/log");
string dbJson = JsonConvert.SerializeObject(globalVarsToSet);

Dictionary<string, string> globalVars = JsonConvert.DeserializeObject<Dictionary<string, string>>(dbJson);

foreach(var variable in globalVars)
{
	project.GVar(variable.Key,variable.Value);
}

```

## Шаг 5 — Проверить работу

Добавь шабонон в Zennoposter и запусти выполнение. 
(если глобальные переменные не появились в Project Maker - запусти первый сниппет шаблона из него)

## Edge cases

**Воркер сменил машину** — HWID изменится. Повтори процедуру с новым HWID. Старый `jvars.dat` работать не будет.

**`GenerateClientBundle` не появляется в Scheduler** — задача не зарегистрирована. Проверь что jVars разблокированы (`Config → Status → isUnlocked: true`).

**`outputFolder` не существует** — папка создаётся автоматически. Если возникает ошибка прав доступа — укажи папку куда z3nIO имеет право записи.

**Нужно обновить `safu.key` у всех клиентов** — если `safu.key` был скомпрометирован, необходимо: (1) сгенерировать новый ключ, (2) перешифровать все кошельки в БД, (3) перегенерировать бандлы для всех клиентов. 

**`safu.key not found`** — файл не найден по пути `.internal\safu.key` рядом с проектом. Проверь что файл лежит в нужной папке и путь правильный.


**Переменная `jVars` не найдена** — убедись что переменная создана в проекте и содержит путь к файлу, а не пустую строку.
