

Живые логи приложения. Показывает записи от всех ZP-задач, которые отправляют логи на `LogHost` из [[shift + ctrl + 0.    Config]].

Путь: `?page=logs`

Автообновление каждые 3 секунды по умолчанию.

---

## Таблица логов

Каждая строка содержит:

|Колонка|Описание|
|---|---|
|Time|Время записи|
|Level|`DEBUG`, `INFO`, `WARNING`, `ERROR`, `SUCCESS`|
|Machine|Имя машины, с которой пришёл лог|
|Project|Имя ZP-проекта|
|Uptime|Время жизни сессии — считается от старта ZP-шаблона. Обновляется в реальном времени|
|PID|ID процесса ZP|
|Port|Порт, через который пришёл лог|
|Account|Аккаунт, с которым работает шаблон в данный момент|
|Origin|Источник лога внутри шаблона|
|Message|Текст сообщения|

Строки с длинным или многострочным сообщением показываются в одну строку. Hover — tooltip с полным текстом. Клик по сообщению — копировать текст в буфер.

Клик по строке — открыть [[#Detail-панель]].

Цвет строки соответствует уровню: ERROR — красный, WARNING — жёлтый, SUCCESS — зелёный.

---

## Фильтры

|Фильтр|Описание|
|---|---|
|Level|`ERROR`, `WARNING`, `INFO`, `SUCCESS`|
|Machine|Конкретная машина из списка|
|Project|Конкретный ZP-проект|
|Session|Конкретная сессия (запуск шаблона)|
|Port|Конкретный порт|
|PID|Конкретный процесс|
|Account|Конкретный аккаунт|
|Origin|Конкретный источник|
|Any Uptime|Фильтр по минимальному времени жизни сессии: 10+ мин, 30+ мин, 1h+, 3h+|
|Лимит|Количество загружаемых записей (10–500, по умолчанию 100)|

Выпадающие списки Machine, Project, Account, Origin, Session, Port, PID заполняются автоматически из текущей статистики сервера.

**↺** — принудительное обновление.

**⏸ Auto ON/OFF** — включить / выключить автообновление.

**✖ Reset** — сбросить все фильтры.

**🗑 Clear** — удалить все логи с сервера (с подтверждением).

## Detail-панель

Открывается при клике на строку. Показывает полную информацию записи.

|Поле|Описание|
|---|---|
|Timestamp|Полная дата и время|
|Level|Уровень с цветовым бейджем|
|Machine|Машина|
|Project|Проект|
|Account|Аккаунт|
|Session|ID сессии|
|Uptime|Время жизни сессии — обновляется в реальном времени|
|PID|ID процесса|
|Port|Порт|
|Origin|Источник|
|Message|Полный текст сообщения|
|Extra|JSON с дополнительными данными (если есть)|

**📋 Copy** — скопировать текст сообщения в буфер.

Закрыть панель — кнопка **×** в заголовке.

## Edge cases

**Логи не появляются** — `LogHost` в [[shift + ctrl + 0.    Config]] не совпадает с адресом, на который ZP-шаблон отправляет логи. Проверь оба значения. Убедись что порт не занят другим процессом — статус виден в `Config → Status → Listening Ports`.

**Uptime показывает число вместо таймера** — session ID не является unix-timestamp. Это нормально для нестандартных шаблонов — uptime просто не считается.

**Фильтр по Session или PID не находит нужного** — выпадающие списки заполняются из агрегированной статистики. Если запись старая и уже вытеснена из памяти — введи значение вручную в поле поиска (не поддерживается в текущем UI, используй фильтр Limit).

**Список устарел после Clear** — нажми **↺** принудительно или дождись автообновления.

**Много записей тормозит браузер** — уменьши лимит или добавь фильтры. При лимите 500 и высокой частоте логирования рекомендуется держать AUTO OFF и обновлять вручную.
