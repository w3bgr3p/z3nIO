

Страница настройки z3nIO. Открывается автоматически при первом запуске, если `appsettings.secrets.json` не найден.


---

## Status

Показывает текущее состояние сервера. Обновляется автоматически раз в секунду.


| Поле            | Описание                                         |
| --------------- | ------------------------------------------------ |
| Uptime          | Время работы с момента запуска                   |
| Port            | Порт дашборда                                    |
| DB Mode         | Режим базы данных: `SQLite` или `Postgre`        |
| DB Host         | Путь к файлу (SQLite) или `host:port` (Postgres) |
| Log Host        | Адрес, на который ZP отправляет логи             |
| Traffic Host    | Адрес, на который ZP отправляет HTTP-трафик      |
| Logs Folder     | Папка хранения лог-файлов                        |

## DbConfig

Подключение к базе данных. Все данные фермы (аккаунты, задачи, прокси) хранятся здесь.

**Выбор режима:**

`SQLite` — один файл на диске, не требует отдельного сервера. Подходит для одной машины. (работа не протестирована)

`Postgre` — внешний PostgreSQL-сервер. Подходит при работе нескольких машин с общей БД.

**Поля SQLite:**

|Поле|Пример|
|---|---|
|SQLite Path|`C:\z3nIO\data\farm.db`|

**Поля Postgres:**

|Поле|Пример|
|---|---|
|PG Host|`localhost`|
|PG Port|`5432`|
|PG Database|`postgres`|
|PG User|`postgres`|
|PG Password|`••••••••`|

После нажатия **💾 Save Config** сервер переподключается к БД без перезапуска.

## Logs & Server

Адреса и папки для приёма логов и трафика от ZP-задач.

|Поле|Описание|Пример|
|---|---|---|
|Dashboard Port|Порт веб-интерфейса|`10993`|
|Log Host|Адрес приёма логов от ZP|`http://localhost:10994`|
|Traffic Host|Адрес приёма HTTP-трафика от ZP|`http://localhost:10995`|
|Logs Folder|Папка для хранения `.jsonl` файлов логов|`./logs`|
|Temp Folder|Временные файлы|`./temp`|
|Reports Folder|Папка отчётов|`./reports`|
|Max File Size (MB)|Мягкий лимит размера одного лог-файла|`50`|

> `Log Host` и `Traffic Host` должны совпадать с адресами, указанными в ZP-шаблоне. Если логи не появляются на странице [[shift + ctrl + 4.    Logs]] — в первую очередь проверяй это поле.

> Изменение `Dashboard Port` требует перезапуска приложения. Остальные поля применяются сразу после Save.

## Security · jVars

Хранилище зашифрованных переменных. Используется для расшифровки данных кошельков в рантайме.

|Поле|Описание|
|---|---|
|Wallet PIN|Пароль шифрования. Не сохраняется в конфиге, нужен только при записи.|
|jVars File Path|Путь к `.dat`-файлу на диске. Пример: `C:\z3nIO\keys\vars.dat`|
|Local vars|Дополнительные key-value записи, сохраняемые в зашифрованном файле|

**Как сохранить:**

1. Введи **Wallet PIN**.
2. Укажи **jVars File Path**.
3. При необходимости добавь записи через **+ Add**.
4. Нажми **🔐 Set PIN & Path**.

Файл шифруется привязкой к hardware ID машины. После сохранения PIN не отображается при загрузке конфига.

> Для добавления новой key-value записи нажми **+ Add**, введи ключ и значение, затем **🔐 Set PIN & Path**.

## Storage

Информация о лог-файлах в `Logs Folder`.

Показывает общий размер, количество файлов и список файлов по убыванию размера. Прогресс-бар показывает заполненность относительно мягкого лимита (`Max File Size × 20`).

**Очистка логов:**

|Кнопка|Что удаляет|
|---|---|
|🗑 Clear App Logs|Логи приложения (`/logs`)|
|🗑 Clear HTTP Logs|HTTP-логи из PM (`/http-logs`)|
|🗑 Clear Traffic Logs|Перехваченный трафик ZP (`/traffic-logs`)|
|☠ Clear ALL Logs|Всё перечисленное выше|

Текущие активные файлы (`current-*.jsonl`) очищаются, но не удаляются. Архивные файлы удаляются полностью.

## Fill DB (Import)

Загрузка данных в базу данных фермы. Кнопка **⬆ Fill DB** открывает мастер импорта.

**Шаг 1** — выбери типы данных для импорта.

**Шаг 2** — вставь данные в текстовое поле и нажми **Import →**.

Доступные типы:

|Тип|Таблица|Описание|
|---|---|---|
|Wallets|`_wlt`|Приватные ключи / seed-фразы. Шифруются через PIN. Типы: `secp256k1` (EVM), `base58` (Solana), `seed` (BIP39)|
|Proxy|`_instance`|Строки вида `ip:port` или `ip:port:user:pass`|
|Addresses|`_addresses`|Публичные адреса кошельков (EVM или Solana)|
|Social|Зависит от платформы|Аккаунты Twitter, Discord, Telegram, GitHub и др. Формат: разделённые строки с маской полей|
|Bio|`_profile`|Никнеймы и bio для профилей|
|Mail|`_mail`|Email-аккаунты: Google, iCloud, Firstmail|
|RPC|`_rpc`|RPC-эндпоинты сетей. Формат: `chain_id;rpc_url;explorer;explorer_api`|
|Deposits|`_deposits`|Депозитные адреса CEX по сети|

> Импорт Wallets требует разблокированного jVars (сервер должен быть запущен с загруженным PIN).

## Edge cases

**Страница открывается при старте вместо Scheduler** — `appsettings.secrets.json` не найден или повреждён. Заполни конфиг и нажми **💾 Save Config**.

**DB Mode переключился, но данные не появились** — при смене SQLite → Postgre база пустая. Данные из старой БД не мигрируют автоматически.

**PIN не помнишь** — файл `.dat` необходимо пересоздать. Пересохрани jVars с новым PIN; старые зашифрованные данные кошельков в БД станут нечитаемы.

**Порт занят** — при запуске z3nIO пропускает занятые порты. Если нужный порт не поднялся — проверь `Config → Status → Listening Ports`. Смени порт в конфиге и перезапусти приложение.
