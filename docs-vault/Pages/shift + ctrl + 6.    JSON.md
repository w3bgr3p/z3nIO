

Интерактивный JSON-viewer с security-детектированием, поиском и replay запросов.

Путь: `/json`

При открытии загружается демо-JSON с примером структуры API-ответов. Вставь свой JSON — демо заменится.

---

## Ввод и парсинг

Левая панель — textarea для ввода JSON.

JSON парсится автоматически при вставке или вводе (с задержкой 300 мс). При объёме больше 500 КБ автопарсинг отключается — нажми **▶ Parse** или `Ctrl+Enter`.

**▶ Parse** — принудительный парсинг.

**{ } Format** — форматировать JSON с отступами (pretty-print).

**✕ Clear** — очистить ввод и дерево.

**⎘ Copy** — скопировать весь текущий JSON в буфер.

**⚡ Replay** — открыть [[#Replay-модальное окно]] для корневого объекта (если он содержит поля `url`, `method`, `requestHeaders`, `requestBody`).

Счётчик символов отображается под textarea. Статистика дерева (keys, depth, rows) — в шапке.

---

## Навигация по дереву

Дерево строится в правой панели с виртуальным скроллом — работает быстро даже на больших JSON (10+ МБ).

**Collapse / Expand:**

- Клик на **▼ / ▶** рядом с узлом — свернуть / развернуть этот узел.
- Кнопки цифр **1 2 3 4...** в шапке — свернуть дерево до указанной глубины.
- **Expand all** — развернуть всё.
- **Collapse all** — свернуть всё.

Hover на строку — путь до узла отображается в строке под поиском.

Строки значений цветом по типу: строки — голубой, числа — оранжевый, булевы — фиолетовый, null — серый.

---

## Поиск

Поле поиска над деревом. Фильтрует узлы по подстроке в ключе или значении. Совпадения подсвечиваются, родительские узлы разворачиваются автоматически.

Счётчик совпадений появляется справа от поля.

---

## Security Badges

На строках с чувствительными данными автоматически появляются цветные бейджи:

|Бейдж|Что детектирует|
|---|---|
|🔑 Bearer|`Authorization: Bearer ...`|
|🔑 Basic|`Authorization: Basic ...`|
|🔑 API Key|`X-Api-Key:` или `X-Auth-Token:`|
|🛡 CF Clear|`cf-clearance` cookie|
|🛡 CF Bot|`__cf_bm`|
|🛡 CF Ray|`CF-Ray:`|
|🛡 Turnstile|`X-Turnstile-Token:` или Turnstile sitekey|
|🤖 reCaptcha|`grecaptcha`, `data-sitekey`|
|🤖 hCaptcha|`hcaptcha.com`|
|🤖 FunCaptcha|`arkoselabs.com`|

Детектирование работает как по ключу, так и по значению — например, строка `"authorization: Bearer abc123"` получит бейдж даже если лежит как значение поля, а не заголовка.

## Действия на узлах

У каждой строки при наведении появляются кнопки:

|Кнопка|Действие|
|---|---|
|**⎘** (copy)|Скопировать значение узла. Для объектов и массивов — JSON с отступами|
|**C#**|Скопировать C# выражение доступа к этому узлу через Newtonsoft `JObject`. Пример: `var token = JObject.Parse(json)["data"]["token"];`|
|**⚡** (replay)|Открыть [[#Replay-модальное окно]] для этого URL (появляется только на строках со значением `https://...` или ключом `url`, `requestUrl`, `endpoint`, `href`)|
|**🚫 / 👁**|Скрыть / показать этот узел|

## Field Filter

Появляется автоматически если JSON содержит массив объектов (например, список запросов из [[shift + ctrl + 5.    HTTP]]).

Панель **Field Filter** показывает все ключи объектов массива по уровням глубины. Каждый ключ — чип с переключателем. Скрытые ключи прячутся во всех элементах массива сразу.

**Пресеты:**

|Пресет|Показывает|
|---|---|
|`api`|url, method, statusCode, requestBody, responseBody|
|`headers`|url, method, statusCode, requestHeaders, responseHeaders|
|`status`|url, method, statusCode|
|`all`|Все поля|

## Hidden Panel

Если есть вручную скрытые узлы — появляется панель над деревом со списком путей скрытых узлов. Клик на путь — вернуть узел обратно. **Show all** — показать все скрытые.

## Replay-модальное окно

Открывается кнопкой **⚡** на строке с URL или кнопкой **Replay** в шапке.

Поля предзаполняются из выбранного узла и его соседей (`method`, `requestHeaders`, `requestBody`).

|Поле|Описание|
|---|---|
|Method|HTTP-метод (редактируемый)|
|URL|Адрес (редактируемый)|
|Headers|`Key: Value`, по одному на строку|
|Body|Тело запроса|

**▶ Send** — отправить запрос через встроенный прокси z3nIO. Результат — статус, время, заголовки и тело ответа.

**📋 Copy response** — скопировать ответ.

**📋 cURL** — скопировать как cURL-команду.

## Edge cases

**JSON не парсится** — в дереве появится сообщение об ошибке с позицией. Проверь JSON через **{ } Format** или внешний валидатор.

**Дерево пустое после вставки** — JSON больше 500 КБ. Нажми **▶ Parse** вручную.

**Security badges не появляются** — детектирование работает только на строковых значениях длиннее 1 символа. Числа, булевы и null не проверяются.

**Replay не работает** — порт `DashboardPort + 1` недоступен. Проверь `Config → Status → Listening Ports`.

**Field Filter не появляется** — JSON не содержит массивов объектов на верхнем уровне.
